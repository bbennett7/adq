05 — dark mode

Dark mode uses forest as the background. Sage remains the accent — same role, just against dark. Sage-light replaces sage for improved contrast. Forest-l is used for subdued text where stone would be too bright. No green backgrounds — sage is always accent only.

06 — usage rules
color rules
  • Sage is an accent color only — never use as a background
  • Forest is the only dark background
  • Stone / stone-2 / stone-3 are the only light backgrounds
  • Use sage-light (not sage) for links and labels on forest backgrounds
  • Rule-l on light bg, rule-d on dark bg — never mix
  • Forest-l appears in: ask wordmark, Caveat asides, footer text on dark, muted labels on dark
typography rules
  • All Fira Code labels are lowercase — no text-transform: uppercase anywhere
  • DM Sans default weight is 300 — use 500 for strong only
  • Caveat is used sparingly — one or two lines max per section
  • Newsreader italic is for archive question lists only
  • Fraunces italic is for the ? in the favicon only
  • Bricolage 700/800 for display, 300/400 for wordmark ask/.ai
component rules
  • Today card is always horizontal (never vertical on desktop)
  • Question and answer labels are top-aligned at the same vertical position
  • Card header always: date (sage) · label (vmuted) · no. (vmuted)
  • Archive rows use Newsreader italic for the question text
  • Field note items always have: tag + title + excerpt + meta
  • Hover state is always opacity .6–.65 — no color changes on hover except arrows
layout rules
  • Max content width: 1200px, side padding: 40px
  • Today card: 2fr question / 3fr answer
  • Below grid: 1fr archive / 320px field notes
  • Start here lives on its own page — not on the landing page
  • About lives on its own page — not on the landing page
  • Nav: date left / wordmark center / links right
